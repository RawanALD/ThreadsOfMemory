# Threads Of Memory – Requirements Document

## 1. Project Overview
Threads Of Memory is a cozy pixel art game that explores Saudi heritage through weaving, storytelling, and memory. The game focuses on delivering an emotional and cultural experience through interactive gameplay and narrative elements.

---

## 2. Project Goal
The goal of this project is to develop a narrative-driven pixel art game that allows players to experience Saudi heritage through interactive weaving mechanics and storytelling.

---

## 3. Project Objectives
- Provide an interactive crafting and weaving system  
- Enable players to collect and manage resources  
- Allow interaction with NPCs to deliver story elements  
- Implement a functional shop system for item exchange  
- Create an immersive environment using a day and night cycle  

---

## 4. Success Criteria
- The core gameplay systems (inventory, crafting, shop) are fully functional  
- The player can complete at least one full gameplay loop (collect, craft, interact)  
- The game runs without major errors or crashes  
- The user interface is responsive and usable  

---

## 5. Scope

### Included
- Single-player experience  
- Crafting and weaving system  
- Inventory management  
- NPC interaction and dialogue  
- Shop system (buy and sell items)  
- Day and night cycle  

### Not Included
- Multiplayer functionality  
- Online features  
- Advanced combat systems  

---

## 6. Functional Requirements

### 6.1 Inventory System
FR-1: The system shall allow the player to view the inventory.  
FR-2: The system shall allow the player to add items to the inventory.  
FR-3: The system shall allow the player to remove items from the inventory.  
FR-4: The system shall allow the player to use items from the inventory.  

### 6.2 Crafting and Weaving System
FR-5: The system shall allow the player to gather materials.  
FR-6: The system shall allow the player to craft items using collected materials.  
FR-7: The system shall display available crafting recipes to the player.  

### 6.3 Shop System
FR-8: The system shall allow the player to purchase items.  
FR-9: The system shall allow the player to sell items.  
FR-10: The system shall display item prices to the player.  

### 6.4 NPC Interaction
FR-11: The system shall allow the player to interact with NPCs.  
FR-12: The system shall provide dialogue and story elements through NPCs.  
FR-13: The system shall allow the player to complete tasks or quests.  

### 6.5 Game System
FR-14: The system shall allow the player to save the game.  
FR-15: The system shall allow the player to load the game.  
FR-16: The system shall include a day and night cycle.  
FR-17: The system shall allow the player to pause the game.  

---

## 7. Non-Functional Requirements
- The game shall run smoothly on standard PC hardware  
- The system shall provide a simple and intuitive user interface  
- The system shall maintain a consistent pixel art style  
- The system shall respond to player input with minimal delay

---

## 8. User Stories
- As a player, I want to collect materials so that I can craft items  
- As a player, I want to interact with NPCs so that I can learn more about the story  
- As a player, I want to manage my inventory so that I can organize my items  
- As a player, I want to buy and sell items so that I can progress in the game  
- As a player, I want to experience a day and night cycle so that the world feels alive

---

## 9. Assumptions
- The game will be developed for PC platforms  
- Players have basic familiarity with keyboard and mouse controls  
- The project is developed by a small team with limited resources  
- The game focuses on a single-player experience only

---

## 10. Constraints
- Development is limited to the Godot Engine and GDScript  
- The project timeline is limited by academic and personal schedules  
- Graphics and assets are limited to available resources or self-created content
